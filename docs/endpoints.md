# Endpoints Reference

All endpoints are mounted under the `/api` prefix (see `main.py`).

| Group | Base path | Source |
| --- | --- | --- |
| Books | `/api/books` | `api/controllers/books.py` |
| Users / Auth | `/api/users` | `api/controllers/users.py` |

---

## Authentication

The API uses **JWT bearer tokens** signed with HS256 (`core/security.py`).

- Obtain a token via `POST /api/users/login`.
- Send it on protected requests as `Authorization: Bearer <token>`.
- Default expiry: **30 minutes** after issue (`ACCESS_TOKEN_EXPIRE_MINUTES`).
- Token payload claims: `sub` (user id), `role`, `iat`, `exp`, `iss = "LookOwl-Server"`.

Endpoints that require auth are marked **Auth required: yes** below.

## Users

### `POST /api/users/register`

Register a new user. Newly created users are assigned the role `LECTOR`.

- **Auth required:** no
- **Request body:** [`RegisterUserDto`](./dtos.md#registeruserdto)
- **Success response (200):**

  ```json
  { "success": true }
  ```

- **Error responses:**

  | Status | When | Body |
  | --- | --- | --- |
  | `409 Conflict` | The user could not be created (e.g. email already exists, persistence failed). | `{ "detail": "User could not be created" }` |
  | `422 Unprocessable Entity` | Body fails DTO validation (see [dtos.md](./dtos.md)). | Pydantic validation error |

---

### `POST /api/users/login`

Validate credentials and issue a JWT bearer token.

- **Auth required:** no
- **Request body:** [`LoginDto`](./dtos.md#logindto)
- **Success response (200):**

  ```json
  {
    "access_token": "<jwt-string>",
    "token_type": "bearer"
  }
  ```

- **Error responses:**

  | Status | When | Body |
  | --- | --- | --- |
  | `401 Unauthorized` | Email not found or password mismatch. | `{ "detail": "User not found" }` |
  | `422 Unprocessable Entity` | Body fails DTO validation. | Pydantic validation error |

## Books

### `GET /api/books/`

Search the catalog with optional filters and cursor-style pagination.

- **Auth required:** no
- **Query parameters** (all optional):

  | Name | Type | Description |
  | --- | --- | --- |
  | `title` | `string` | Substring/title filter applied by the repository. |
  | `author` | `string` | Author filter applied by the repository. |
  | `limit` | `int` | Maximum number of books to return in this page. Internally the controller asks the service for `limit + 1` so it can detect whether a next page exists. |
  | `offset` | `int` | Number of records to skip (used together with `limit`). |

- **Success response (200):**

  ```json
  {
    "data": [
      {
        "id": 1,
        "title": "El Aleph",
        "isbn": "9788499890944",
        "description": "...",
        "editorial": "Debolsillo",
        "publication_date": "2011-05-19",
        "cover_url": "https://example.com/cover.jpg",
        "language": "es",
        "author": ["Jorge Luis Borges"],
        "category": ["Cuento", "Fantástico"],
        "page_count": 224
      }
    ],
    "page": {
      "next_cursor": { "offset": 20 },
      "has_next": true
    }
  }
  ```

  - `data` is a list of `Book` domain objects (see [dtos.md → Domain: Book](./dtos.md#domain-models-returned-by-the-api)).
  - `page.has_next` is `true` when more results exist past the requested `limit`.
  - `page.next_cursor.offset` is the offset the client should use for the next page (`limit + offset` when `offset` was supplied, otherwise `limit`). It is `null` if there is no next page.

- **Error responses:**

  | Status | When |
  | --- | --- |
  | `422 Unprocessable Entity` | A query parameter cannot be parsed (e.g. non-integer `limit`). |

---

### `POST /api/books/register`

Register a new book in the catalog.

- **Auth required:** **yes** (bearer token; user is extracted via `core.security.extract_user`).
- **Request body:** [`RegisterBookDto`](./dtos.md#registerbookdto)
- **Success response (200):**

  ```json
  { "id": 42 }
  ```

  `id` is the autogenerated identifier of the newly created book.

- **Error responses:**

  | Status | When | Body |
  | --- | --- | --- |
  | `401 Unauthorized` | Missing/invalid bearer token, or token does not resolve to a user. | `{ "detail": "Invalid token or user" }` |
  | `409 Conflict` | ISBN already exists, or other DB integrity / SQLAlchemy error. | `{ "detail": "ISBN already exists" }` or `{ "detail": "Unknown exception" }` |
  | `422 Unprocessable Entity` | Body fails DTO validation (e.g. invalid ISBN checksum, empty title, non-positive `page_count`, malformed `cover_url`, `language` not exactly 2 chars). | Pydantic validation error |

---

## Status code summary

| Code | Meaning in this API |
| --- | --- |
| `200 OK` | Successful response (the API does not currently use `201 Created`). |
| `401 Unauthorized` | Authentication failed (missing/invalid token or wrong credentials). |
| `409 Conflict` | Resource could not be created due to a conflict (duplicate ISBN, duplicate user). |
| `422 Unprocessable Entity` | Validation error reported by Pydantic / FastAPI. |

## Endpoint inventory at a glance

| Method | Path | Auth | Purpose |
| --- | --- | --- | --- |
| `GET`  | `/api/books/` | no | List/search books with pagination |
| `POST` | `/api/books/register` | yes | Create a new book |
| `POST` | `/api/users/register` | no | Create a new reader account |
| `POST` | `/api/users/login` | no | Issue a JWT bearer token |

Endpoints for `ejemplares` (book copies) and `prestamos` (loans) are not yet exposed; the domain models, repositories, and DB tables exist but no controller routes have been published.
